# Notion Sync Protocol

## Purpose
Keep the AI OS workspace aligned with actual implementation progress.

## Required updates during kickoff
- Ensure a project record exists for Konative.com
- Ensure a site registry record exists for konative.com
- Ensure milestone records exist for kickoff phases
- Ensure first 14 blocks exist in Block Library with planned status

## Update cadence
- After each milestone completion, update:
  - project status or phase,
  - completed tasks,
  - milestone progress,
  - any new block records,
  - repo/live/admin URLs when available.

## Logging convention
When making a Notion update, log:
- what database/page changed,
- what record was created or updated,
- the purpose of the change.

## Safety rules
- Do not create duplicate projects if a record already exists.
- Search before creating.
- If multiple possible matches exist, stop and surface the ambiguity.
